# 参考内容（Demo）

很多人以为企业 agent 真正的瓶颈还在模型能力本身。

但一旦它真的进入企业现场，问题就会迅速切到另一层：
- 它能不能拿到对的上下文
- 它能不能读到对的知识
- 它有没有被允许碰到真实系统

也就是说，企业 agent 的难点，往往不是“会不会回答”，而是“能不能接进工作现场”。

这会直接改变判断标准：
以后看一个企业 agent 项目，不该只看模型够不够强，而应该优先看：
1. 上下文是怎么组织和注入的
2. 知识更新怎么做
3. 权限边界怎么控制
4. 它在真实流程里承担哪一段职责

如果这四件事说不清，模型再强，最后也可能只是一个 demo。

如果这四件事能说清，哪怕模型不是最强的，也更有机会真正上岗。

这也是为什么很多企业自动化项目，越往后看，越像是在比“上下文接入能力”。
