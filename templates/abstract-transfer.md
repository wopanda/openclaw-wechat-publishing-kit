# 抽象层迁移方法论

> 日新专有功能：当用户提供参考文章时，提取爆款文章的底层逻辑结构，用「日新」的角色和内容重新演绎。
> 标准流程（无参考文章时）直接用 SKILL.md 中的执行流程即可。

---

## 核心原则

- **抽象层迁移，不是内容改写**：学习「为什么这篇文章火」的底层逻辑，而不是改写内容
- **结构可复用，内容必原创**：叙事结构、情绪设计、论证逻辑可借鉴，但所有具体内容（案例、数据、表达）必须是自己的
- **保持人设一致性**：即使借鉴了爆款结构，最终呈现的必须是「日新」的声音和视角

---

## 执行步骤

### 第一步：深度解构参考文章

**提取叙事结构**：
- 开头用了什么钩子？（数据冲击 / 反常识观点 / 故事场景 / 痛点共鸣）
- 中间如何展开论证？（案例堆叠 / 对比反差 / 层层递进 / 问题-解决）
- 结尾如何升华？（金句总结 / 行动召唤 / 情感共鸣）

**识别情绪曲线**：
- 开头制造什么情绪？（好奇 / 焦虑 / 共鸣 / 惊讶）
- 中间如何维持注意力？（反转 / 冲突 / 悬念 / 价值感）
- 结尾留下什么感受？（启发 / 行动欲 / 期待 / 认同）

**分析论证逻辑**：
- 核心观点是什么？（一句话概括）
- 用了哪些论据类型？（数据 / 案例 / 对比 / 类比 / 引用）
- 逻辑链条是什么？（因果 / 递进 / 并列 / 转折）

**捕捉语言特征**：
- 金句在哪里出现？（开头 / 小节开头 / 结尾）
- 用了什么修辞？（排比 / 对比 / 反问 / 比喻）
- 节奏如何控制？（长短句搭配 / 段落密度）

### 第二步：迁移到「日新」人设

**保留结构骨架**：
- 使用相同的叙事结构（如：痛点开头 → 3个案例 → 方法论 → 行动召唤）
- 复制情绪曲线设计（如：焦虑 → 启发 → 信心 → 行动）
- 借鉴论证逻辑（如：现象 → 本质 → 方法 → 未来）

**替换内容填充**：
- 开头钩子 → 用 AI 领域的数据/现象/痛点替换
- 案例素材 → 用数字员工实践案例或 AI 工具案例替换
- 核心观点 → 转化为 AI 应用领域的对应观点
- 金句表达 → 保持修辞手法，用 AI 主题语言重写

**融入「日新」特色**：
- 加入技术背景的独特视角（如：从开发者角度看问题）
- 植入个人实践经历（如：做数字员工时的真实感悟）
- 保持务实真诚的语言风格（避免原文可能的夸张或煽动）

### 第三步：原创化处理

- **语言层面**：所有句子用自己的话重写，不保留原文的具体表达
- **案例层面**：100% 使用不同的案例，优先用真实实践
- **数据层面**：使用 AI 领域的真实数据，不复制原文数据
- **观点层面**：在原文观点基础上，加入独特思考和延伸

### 第四步：质量检查

- 文章读起来是「日新」的原创思考，而非改写
- 核心价值点清晰，不因迁移而模糊
- 案例和数据真实可查，不因套用结构而编造
