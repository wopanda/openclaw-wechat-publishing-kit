# 飞书图文报告版式骨架

当输出目标是“在飞书里点开就能直接看”的图文报告时，不要把内容写成普通长文。
应优先组织成适合飞书阅读的短段落、清晰层级、图文交替节奏。

## 第一性原则

1. 飞书里的阅读是扫读，不是沉浸式长篇阅读。
2. 每一屏都要让人快速知道：这部分在说什么、为什么值得看。
3. 图文切换要服务理解，不要让图片变成无意义插图。
4. 标题、导语、分节、图位、结尾总结，必须形成明确节奏。

## 推荐骨架

```md
# 主标题

> 一句话导语 / 判断句

### 图片位｜cover_01｜封面头图
- ...

## 一、问题是什么
短导入。

## 二、为什么原来的做法不够了
2-4 个短段落。

### 图片位｜sec2_compare_01｜前后对比图
- ...

## 三、现在应该怎么重组
2-5 个短段落，给方法和结构。

### 图片位｜sec3_method_01｜方法示意图
- ...

## 四、证据 / 案例 / 结果
2-4 个短段落。

### 图片位｜sec4_evidence_01｜证据图
- ...

## 五、结论
用 1-3 个短段落收口。
```

## 段落规则

- 单段尽量短，优先 1-3 句。
- 尽量避免超长大段。
- 如果一段承担转折，最好单独成段。
- 如果一段承担判断，最好先给判断句，再补解释。

## 图文切换规则

### 适合放图的位置
- 标题后：建立主题感
- 关键转折后：帮助理解结构变化
- 案例后：把证据可视化
- 结尾前：做总结或收束

### 不适合放图的位置
- 纯叙述已经很顺的地方
- 没有明确表达任务的地方
- 只为了“看起来像图文”而硬塞的地方

## 标题规则

### 主标题
- 要像一份报告或图文稿标题，而不是论文标题
- 优先体现判断、变化、结果、问题

### 二级标题
- 优先用清楚的人话
- 尽量让读者只看标题就知道这一段为什么存在

## 结尾规则

- 不要拖成长总结
- 优先收在一个判断、一个提醒、或一个动作建议上
- 如果全文强调“重组”，结尾就要回到“为什么这样重组后更可读、更可用、更可传播”

## 输出检查清单

交付前至少检查：

1. 主标题是否足够像报告标题
2. 导语是否在 1 句话内说清核心判断
3. 图位是否都承担明确任务
4. 每节是否都能单独被扫读理解
5. 结尾是否收在判断，而不是停在描述
