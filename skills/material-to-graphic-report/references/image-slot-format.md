# 结构化图片位格式

当任务目标是图文报告、飞书图文稿、可后续补生图的视觉化内容时，统一使用下面的图片位结构。

## 使用原则

1. 每张图都要有明确作用：封面、解释、对比、证据、转折、收口。
2. 不要只写“这里配图”或“放一张图”。
3. 当前没有生图能力时，图片位也必须完整写出，供后续逐位生成与替换。
4. 图片位要可直接插入 Markdown / 飞书文档原型中。

## 标准模板

```md
### 图片位｜<slot_id>｜<图片名称>
- 状态：待生成
- 用途：这张图要解决什么表达任务
- 位置：封面 / 第X节前 / 第X节论点后 / 结尾收口处
- 画面描述：这张图应该让人看到什么
- 生图提示词：可直接给生图模型的提示词
- 比例：16:9 / 4:3 / 1:1 / 9:16
- 风格：信息设计感 / 商务科技感 / 信息图 / 纪实截图风 / 极简高级
- 图注：放在图下方的一句话说明
```

## 推荐字段解释

### `slot_id`
使用稳定、可回填的命名方式，例如：
- `cover_01`
- `sec2_img1`
- `sec3_compare_01`

### `用途`
优先从以下类型里选择：
- 封面建立主题
- 解释抽象概念
- 展示前后对比
- 承接证据或数据
- 做段落转折
- 做结尾收口

### `位置`
不要写模糊位置。优先写成：
- 封面标题下方
- 第二节开头
- 第二节核心论点之后
- 案例段落之后
- 结尾总结前

### `画面描述`
用人能理解的语言描述“这张图应该长什么样”，不要只丢风格词。

### `生图提示词`
尽量包含：
- 主体场景
- 画面关系
- 风格
- 色调
- 构图
- 比例

## 示例1：封面图

```md
### 图片位｜cover_01｜封面头图
- 状态：待生成
- 用途：建立主题感，让读者一眼知道这是从杂乱素材压成图文报告的内容
- 位置：封面标题下方
- 画面描述：深色背景上的分散信息流汇聚成一份被高亮的报告
- 生图提示词：科技感、信息汇聚、深色背景、报告高亮、简洁高级、适合飞书封面、16:9
- 比例：16:9
- 风格：商务科技感 / 简洁高级
- 图注：把分散素材压成一份可读报告
```

## 示例2：前后对比图

```md
### 图片位｜sec2_compare_01｜旧结构 vs 新结构
- 状态：待生成
- 用途：展示重组前后的表达差异
- 位置：第二节核心论点之后
- 画面描述：左侧是碎片化内容，右侧是被重新编排成清晰模块的图文结构
- 生图提示词：前后对比、信息重组、模块化、结构优化、简洁信息图风格、横版、4:3
- 比例：4:3
- 风格：信息图 / 结构对比图
- 图注：不是换词，而是重组表达结构
```

## 示例3：证据承接图

```md
### 图片位｜sec3_evidence_01｜证据卡片图
- 状态：待生成
- 用途：承接数据或关键事实，让正文证据可视化
- 位置：第三节案例段之后
- 画面描述：以卡片式呈现 3 条关键证据，强调结论支撑
- 生图提示词：信息卡片、数据证据、简洁排版、重点高亮、专业报告风、1:1
- 比例：1:1
- 风格：信息卡片 / 报告风
- 图注：关键证据不是堆材料，而是为结论服务
```
