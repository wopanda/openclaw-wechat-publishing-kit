# 图文报告输出模板

当使用 `material-to-graphic-report` 时，优先按下面的五层结构输出。除非用户明确要求极简版本，否则不要只返回单层草稿。

---

## 第一层：素材拆解

```md
# 素材拆解

## 这批素材为什么可用
- 素材A：可借什么
- 素材B：可借什么
- 素材C：可借什么

## 可复用结构
- 原素材中最值得保留的结构骨架是什么
- 哪个段落推进方式最有效
- 哪类证据最适合被继承

## 可复用证据 / 案例
- 证据1：可用于支撑什么判断
- 证据2：可用于支撑什么判断
- 案例1：适合放在哪一节

## 不应直接沿用的部分
- 哪些 framing 不该照搬
- 哪些表述太旧 / 太散 / 太像原文
- 哪些部分会把稿子带偏
```

---

## 第二层：重组方案

```md
# 重组方案

## 新角度
一句话说清这篇图文报告最终要成立什么判断。

## 新标题候选
1. 标题候选一
2. 标题候选二
3. 标题候选三

## 新提纲
### 一、这一页先讲什么
### 二、为什么旧结构不够了
### 三、现在应该怎么理解 / 怎么做
### 四、证据 / 案例 / 结果
### 五、结论 / 收口

## 新论证顺序
- 为什么先讲A，再讲B
- 为什么图要插在这些位置
- 为什么结尾要收在这个判断上
```

## 第五层：交付层

```md
# 交付层

- Markdown：已完成
- Feishu 图文原型：已完成 / 未创建
- 图片位数量：4
- 当前图片状态：占位待生成 / 已生成待替换 / 已替换
- 后续替换方式：按 slot_id 逐位回填
- 替换协议：遵循 `references/image-replacement-protocol.md`
```

## 极简模式（仅在用户明确要求快时使用）

如果用户明确要求“先快出一版”，可以压成三层：

1. 新角度 + 新提纲
2. 飞书图文稿
3. 图片位清单

但即使是极简模式，也不要省略图片位定义。
