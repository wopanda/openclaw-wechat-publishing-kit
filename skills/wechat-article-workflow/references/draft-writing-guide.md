# Workflow Draft Writing Guide

## 目标
把 `workflow context + 选题卡 + 表达卡` 压成一篇干净正文。

## 正文必须做到
1. 像一篇可以继续进入预览或发布链的文章
2. 优先保住用户长期主线、真实经历、失败教训、个人判断
3. 不暴露执行过程、脚本过程、检索过程
4. 不把标题候选、图像提示词、交付备注混进正文
5. 外部样板只增强表达，不覆盖用户自己的口吻
6. 默认优先写清冲突、代价、误解或反常识，不写空泛正确话

## 正文不要包含
- “根据你提供的资料”
- “我读取了项目文档”
- “下面是选题卡/表达卡”
- “如果需要我可以继续”
- 工作流说明、操作说明、脚本输出

## 正文建议结构
1. 开头先破误解 / 设冲突
2. 中间拆真实卡点与底层逻辑
3. 用真实经历或失败教训做锚点
4. 结尾给出更稳的判断与收口

## 开头写法要求
- 开头要有判断感，但不要固定成同一句式反复使用。
- 优先根据题目选择开头方式：
  - 先抛判断
  - 先抛误解
  - 先抛场景
  - 先抛反常识事实
  - 先抛问题
- `我后来越来越确定……` 只在经验判断型 / 认知修正型 / 复盘型文章中按需使用。
- 不要让开头像模板函数输出，要让它像一个活人在进入这篇文章。
