# Context Loading Contract

## 目标
把 `Projects/Active/微信公众号文章自动化发布` 里的规则与个人上下文，稳定压成一份最小工作上下文，供后续选题卡、表达卡与正文起草使用。

## 固定读取顺序
1. 流程真相源
2. 个人上下文库
3. 样板库地图
4. 当前任务输入
5. 按需补运行记录

## 最小必读文件
- `00-主线与流程/17-默认执行链-v2.md`
- `00-主线与流程/16-公众号内容质量门-v1.md`
- `02-个人上下文库/01-主线与价值观.md`
- `02-个人上下文库/02-真实项目经历.md`
- `02-个人上下文库/03-失败教训.md`
- `02-个人上下文库/04-表达偏好.md`
- `02-个人上下文库/05-人物定位.md`
- `03-样板库/02-外部样板/01-外部样板使用地图-v1.md`

## 主题入口默认取料顺序
1. 主线与价值观
2. 真实项目经历
3. 失败教训
4. 表达偏好 / 人物定位
5. 历史运行记录
6. 外部材料

## 注意
- 只读最小必要文件，不要把整个 Projects 全量塞进上下文
- 运行记录必须按需读，不是默认全读
- 上下文来自项目目录，不要把真相长期硬编码进 skill
