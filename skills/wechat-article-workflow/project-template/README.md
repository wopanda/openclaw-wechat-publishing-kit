# 当前技能包演示项目模板

这个目录不是你的真实生产项目，只是一个**可开箱演示骨架**。

如果你要真正跑 `wechat-article-workflow`：

## 方式 A：直接用这个模板补内容
按下面结构补自己的文件：
- `00-主线与流程/`
- `02-个人上下文库/`
- `03-样板库/`
- `04-运行记录/`

## 方式 B：指向你自己的项目目录
运行脚本或使用 skill 时，传入：
- CLI: `--project-root /your/project/path`
- 或环境变量：`WECHAT_ARTICLE_PROJECT_ROOT=/your/project/path`

## 目录说明
- `00-主线与流程/`：规则与流程门禁
- `02-个人上下文库/`：人设、经历、失败教训、表达偏好
- `03-样板库/`：可借结构的样板
- `04-运行记录/`：观点卡、证据包、已有成稿候选

如果这些文件不存在，脚本会把它们标记为缺失，但不会再假定必须在 `/root/obsidian-vault/...` 才能运行。
