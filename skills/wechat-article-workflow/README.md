# wechat-article-workflow 稳定化改造说明

这次改造的目标不是新增一个大而全的写作器，
而是把 `wechat-article-workflow` 收成一个真正稳定的主调度 skill。

## 已补的关键点

### 1. 补清了输入入口
现在明确保留 4 个入口：
- 日报选点
- 给一篇文章
- 给一个主题
- 给一批材料 / 线索

### 2. 补清了必读文件
现在明确区分：
- 流程真相源
- 个人上下文库
- 样板库
- 运行记录
- 发布链附加规则

并强调“只读最小必要文件”，避免过度耦合。

### 3. 补清了主题入口的默认取料顺序
当用户只给一个主题时，默认先读：
1. 主线与价值观
2. 真实项目经历
3. 失败教训
4. 表达偏好 / 人物定位
5. 历史运行记录
6. 外部材料

### 4. 补清了标准执行顺序
统一主链：
- 收口输入
- 选题卡
- 观点菜单
- 用户选观点
- 真实经历调用
- 生成 draft handoff bundle
- 由当前主模型 / 当前 Agent 起草正文
- 过质量门
- 按需进入预览 / 草稿箱

并补充固定口径：
- `wechat-article` 只当抓文 / 拆样板 / 提参考内容的辅助能力
- 不把外部作者风格写死成默认人格
- 标题优先抓冲突、代价、误解、反常识，不套绝对公式
- 封面与配图判断后置到发布前状态机，不前置为写作必经步骤

### 5. 补清了与后链 skill 的边界
- `material-to-graphic-report`：只负责飞书预览 / 视觉脚本 / 图文原型
- `wechat-draft-publisher`：只负责发布前清洗 / 预检 / 草稿箱
- `wechat-article-workflow`：只负责内容前链调度

### 6. 补清了必须停住的情况
包括：
- 素材不够
- 没有真实经历支撑
- 表达卡未定
- 质量门不通过
- 图片 / 发布链阻塞

## 当前效果

现在 `wechat-article-workflow` 的定位已经更稳定：

> 不是新的写作器，
> 也不是新的发布器，
> 而是先按项目目录读取规则，再把一条线索推进成可写内容的主调度 skill。
