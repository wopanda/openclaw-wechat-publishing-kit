# 去 AI 味 / 人性化收口规则（从 5号 写作工位提炼）

这套规则不是“把稿子改花”，而是把明显的模板腔、机器腔、口水腔压下去。

## 核心目标

- 让文字更像真人在说话
- 保留事实、判断、案例，不靠花哨辞藻掩盖内容空洞
- 服务于发布前最后一轮收口，而不是替代写作本身

## 要重点识别的高频 AI 骨架

### 1. 顺序脚手架
高频信号：
- `首先`
- `其次`
- `再次`
- `最后`

处理思路：
- 能删就删
- 能并回自然句就并回
- 除非确实在做明确分层，否则不要机械排队列

### 2. 填充转场
高频信号：
- `值得注意的是`
- `需要指出的是`
- `可以看到`
- `不难发现`

处理思路：
- 去掉这些“说话前的铺垫”
- 直接说结论或直接上事实

### 3. 总结腔
高频信号：
- `综上所述`
- `总而言之`
- `总的来说`

处理思路：
- 除非真在全文收口，否则优先去掉
- 用更短、更自然的收尾句替代

### 4. 抽象框架词
高频信号：
- `在……方面`
- `在……层面`
- `具有重要意义`
- `带来了深远影响`

处理思路：
- 把抽象概念改成具体动作、具体结果、具体场景
- 能写“做了什么 / 省了什么 / 多了什么”，就不要写宏大判断

### 5. 空泛拔高
高频信号：
- `本质上`
- `从某种意义上说`
- `具有深远意义`
- `值得我们深入思考`

处理思路：
- 没有新增信息就删
- 只保留能承载真实判断的句子

### 6. 句段节奏过于平均
高频信号：
- 每句长度都差不多
- 每段结构都很整齐
- 排比 / 对仗太多

处理思路：
- 打散节奏
- 让句子长短有自然变化
- 把抽象总结句换成现场画面或具体动作

## 正确做法

- 删掉填充短语
- 打散过分工整的排比和模板化转场
- 把抽象概念改成具体动作、场景、案例
- 收成自然、克制、讲人话的表达

## 明确边界

这一步：
- **能做**：降机器感、降模板感、收紧口气
- **不能做**：替内容造血、伪造事实、补充用户没给的观点

如果原稿：
- 事实不够
- 观点不真
- 案例不实
- 结构不成立

那再顺的句子也不能算好稿。

## 对 5.2 最稳的接法

在这条仓库里，这套规则只作为：

`已成型稿件 -> 去 AI 味 / 人性化收口 -> 发布预检 -> 草稿箱`

不要把它重新扩成 5.1 风格的写作总链。
